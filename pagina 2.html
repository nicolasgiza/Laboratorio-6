Página 2: Comandos Básicos de Git

git config --global user.name "Tu Nombre" - Establece el nombre de usuario para Git.

git config --global user.email "tuemail@example.com" - Establece el email para Git.

git config --list - Muestra la configuración actual de Git.

git init - Inicializa un repositorio Git.

git clone URL - Clona un repositorio remoto.

git status - Muestra el estado actual del repositorio.

git add archivo - Añade archivos al área de preparación.

git commit -m "mensaje" - Confirma los cambios en el repositorio.

git push - Envía tus cambios al repositorio remoto.

git pull - Obtiene los cambios del repositorio remoto.

git fetch - Descarga contenido del repositorio remoto sin fusionarlo.

Ciclo de Vida de los Archivos en Git

En Git, los archivos pueden estar en cuatro estados diferentes: sin seguimiento (untracked), modificados (modified), preparados (staged) y confirmados (committed). Entender este ciclo de vida es fundamental para usar Git de manera efectiva.

Cuando creas un nuevo archivo en tu repositorio, este comienza sin seguimiento. Git no lo monitoreará a menos que le digas explícitamente que lo haga con git add. Una vez añadido, el archivo está en el área de preparación, listo para ser confirmado.

Después de hacer modificaciones a un archivo rastreado, necesitas prepararlo nuevamente con git add antes de poder confirmar los cambios. Finalmente, cuando haces un commit con git commit, los cambios se guardan en la base de datos de Git.
